# Adopters

Adopter-specific mappings belong here and must stay out of the core protocol spec.

These materials are non-normative.

Use this area to document:

- how an existing system maps to `AGRP v1`
- which protocol concepts have local equivalents
- where local gaps, deviations, or extensions remain

Do not use this area to redefine protocol semantics or to imply that one adopter architecture is required by the standard.

Included material:

- `templates/adopter-mapping-template.md`: reusable mapping template for design partners and adopters
- `examples/generic-adopter-mapping-example.md`: neutral filled example showing how to use the template without privileging a real product
