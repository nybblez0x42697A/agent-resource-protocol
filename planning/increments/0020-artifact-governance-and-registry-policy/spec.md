# Feature: Artifact Governance and Registry Policy

## Overview

Define governance rules for protocol artifact identifiers, registry curation boundaries, delegation, and reservation policy.

## User Stories

### US-001: Define artifact governance and registry rules
**Project**: agent-resource-protocol

**As a** protocol implementer or steward
**I want** clear governance and identifier registry rules
**So that** profiles, extensions, bindings, and related artifacts can be curated consistently without confusing central authority with implementation ownership

**Acceptance Criteria**:
- [x] **AC-US1-01**: The repo defines baseline governance rules for artifact identifiers, reservation, delegation, and publication boundaries
- [x] **AC-US1-02**: The repo defines what is normative registry policy versus what remains implementation- or ecosystem-specific
- [x] **AC-US1-03**: The repo includes non-normative examples showing a reserved identifier declaration and a delegated namespace policy
