# Feature: Conformance and Examples

## Overview

Define a baseline conformance model and add non-normative example payloads that align with the current protocol schemas.

## User Stories

### US-001: Define baseline conformance and examples
**Project**: agent-resource-protocol

**As a** protocol implementer
**I want** clear conformance expectations and worked example payloads
**So that** I can validate implementations and understand the intended protocol shapes

**Acceptance Criteria**:
- [x] **AC-US1-01**: The repo defines a baseline conformance model for current protocol artifacts
- [x] **AC-US1-02**: The repo includes non-normative example payloads for core records and transitions
- [x] **AC-US1-03**: The examples align with the current schema set without introducing new semantics
