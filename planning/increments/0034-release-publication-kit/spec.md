# Feature: Release Publication Kit

## Overview

Add a non-normative publication kit for `AGRP v1.0.0` so the protocol can be released repeatably without inventing a mandatory packaging or hosting mechanism.

## User Stories

### US-001: Publish a reusable AGRP release kit
**Project**: agent-resource-protocol

**As a** protocol maintainer or release publisher
**I want** a checklist and example package layout for `AGRP v1.0.0`
**So that** I can publish the release consistently while preserving the normative boundary and avoiding ad hoc omissions

**Acceptance Criteria**:
- [x] **AC-US1-01**: The repo adds a non-normative release publication checklist for `AGRP v1.0.0`
- [x] **AC-US1-02**: The repo includes an example publication manifest or package layout that points to the required normative anchors
- [x] **AC-US1-03**: The release kit clearly states that it is illustrative and does not replace the normative release-definition and artifact-set documents
