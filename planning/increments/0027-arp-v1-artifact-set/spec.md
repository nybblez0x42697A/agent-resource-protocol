# Feature: ARP v1 Artifact Set

## Overview

Define the normative artifact set that constitutes `ARP v1` so baseline conformance and future release/version decisions can anchor to one explicit document bundle.

## User Stories

### US-001: Define the ARP v1 artifact set
**Project**: agent-resource-protocol

**As a** protocol author, implementer, or evaluator
**I want** one normative definition of which artifacts make up `ARP v1`
**So that** conformance and future versioning decisions refer to a stable published standard boundary instead of an implicit document set

**Acceptance Criteria**:
- [x] **AC-US1-01**: The repo defines the normative documents that make up `ARP v1`
- [x] **AC-US1-02**: The repo distinguishes core `v1` artifacts from non-normative examples and deferred or future areas
- [x] **AC-US1-03**: The repo states how `ARP v1` conformance claims are anchored to the defined artifact set
- [x] **AC-US1-04**: The repo includes a non-normative example manifest for the `ARP v1` artifact set
