# Examples

Non-normative examples belong here.

- `minimal-resource-registry/`: smallest useful registry example
- `reflection-loop/`: example self-evolution loop
- `sample-wire-format/`: illustrative transport-neutral message examples
- `http-json-binding/`: illustrative HTTP request and response examples for the baseline binding
- `security-policy/`: illustrative denial examples for authorization and policy enforcement
- `extensions-profiles/`: illustrative extension and stricter profile declarations
- `capability-discovery/`: illustrative capability advertisements and negotiation outcomes
- `versioning-policy/`: illustrative additive and breaking evolution examples
- `deprecation-policy/`: illustrative deprecation notices and sunset completions
- `dependency-composition/`: illustrative required and optional dependency compositions
- `failure-diagnostics/`: illustrative compatibility and bounded internal failure diagnostics
- `observability-traces/`: illustrative correlated request and failure reference examples
- `rollout-policy/`: illustrative preview and general availability stage examples
- `artifact-governance/`: illustrative reserved identifiers and delegated namespaces
- `readiness-profiles/`: illustrative operational readiness profile declarations
- `profile-adoption/`: illustrative readiness profile revision and in-progress adoption declarations
- `profile-interoperability/`: illustrative interoperable readiness declarations and discovery support-state advertisements
- `profile-resolution/`: illustrative declaration conflicts and invalid supersession-chain outcomes
- `compliance-integration/`: illustrative integrated compliance evaluation and precedence outcomes
- `compliance-partial-failures/`: illustrative mixed compliance outcomes with invalid declarations or incomplete evidence
